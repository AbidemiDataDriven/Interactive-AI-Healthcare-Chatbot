3. Data Loading & Cleaning

The dataset is structured as a binary feature matrix: each row represents a patient scenario, each column represents a symptom (0 or 1), and the final column is the disease label (prognosis). 

training = pd.read_csv("Data/Training.csv")
testing  = pd.read_csv("Data/Testing.csv")

# Strip trailing digits from duplicate column names
training.columns = training.columns.str.replace(r"\d+$", "", regex=True)
testing.columns  = testing.columns.str.replace(r"\.\d+$", "", regex=True)

# Remove any duplicate columns that remain
training = training.loc[:, ~training.columns.duplicated()]
testing  = testing.loc[:,  ~testing.columns.duplicated()]

A subtle but critical cleaning step: CSV exports sometimes append numeric suffixes to duplicated column names (symptom, symptom.1, symptom2). The regex r"\d+$" strips those trailing digits, and then ~duplicated() keeps only the first occurrence of any column name.

Feature-Target Split

cols = training.columns[:-1]   # all columns except 'prognosis'
X    = training[cols]
y    = training["prognosis"]

# Encode string labels to integers
le = preprocessing.LabelEncoder()
y  = le.fit_transform(y)
💡 Why LabelEncoder?

Random Forest works with integer targets, not string labels. LabelEncoder maps each unique disease name to a number (alphabetically). We store the encoder (le) so we can inverse transform predictions back to disease names later.

X_train, X_test, y_train, y_test = train_test_split(

model = RandomForestClassifier(n_estimators=300, random_state=42)

Why 300 trees? More trees generally improve stability and reduce variance at the cost of training time. For a dataset this size, 300 is a practical sweet spot — fast enough to train in seconds, robust enough to generalize. You can experiment with n_estimators=100 or 500 and compare using model.score(X_test, y_test).

5. The NLP Symptom Extractor — The Hardest Part

This is where the project gets genuinely interesting. Real users don't type stomach_pain — they type "my tummy hurts" or "I have a belly ache". The extractor bridges that gap using three cascading strategies:

1

Synonym Mapping

A hand-curated dictionary maps common lay terms to standardized symptom names. "Loose motion" → diarrhea. "Feaver" → fever. "Shortness of breath" → breathlessness. This handles the most common colloquialisms.

2

Exact Substring Match

After synonym replacement, the cleaned text is scanned for exact occurrences of known symptom names (underscores replaced with spaces). This catches any symptom the user happened to phrase correctly.

3

Fuzzy Matching via difflib

Each word in the user's input is compared to every known symptom using Levenshtein-like distance. A cutoff of 0.8 means the word must be at least 80% similar. This catches typos like "headace" → headache.

def extract_symptoms(user_input, all_symptoms):
    extracted = []
    text = user_input.lower().replace("-", " ")

    # Pass 1: Synonym replacement
    for phrase, mapped in symptom_synonyms.items():
        if phrase in text:
            extracted.append(mapped)

    # Pass 2: Exact match on standardized names
    for symptom in all_symptoms:
        if symptom.replace("_", " ") in text:
            extracted.append(symptom)

    # Pass 3: Fuzzy match for typo tolerance
    words = re.findall(r"\w+", text)
    for word in words:
        close = get_close_matches(
            word,
            [s.replace("_", " ") for s in all_symptoms],
            n=1, cutoff=0.8
        )
        if close:
            for sym in all_symptoms:
                if sym.replace("_", " ") == close[0]:
                    extracted.append(sym)

    return list(set(extracted))  # deduplicate
🔬 Under the Hood: difflib.get_close_matches

Python's difflib uses the Ratcliff/Obershelp algorithm — it finds the longest common substring recursively and computes a similarity ratio. It's fast, requires no installation, and works surprisingly well for symptom-style short words.

6. Encoding Symptoms as Feature Vectors

The model was trained on a 132-dimensional binary feature space. For inference, we need to convert any list of symptom strings back into that exact same vector format: 

# Build a lookup: symptom name → column index
symptoms_dict = {symptom: idx for idx, symptom in enumerate(X)}

def predict_disease(symptoms_list):
    # Start with all zeros
    input_vector = np.zeros(len(symptoms_dict))

    # Flip the relevant indices to 1
    for symptom in symptoms_list:
        if symptom in symptoms_dict:
            input_vector[symptoms_dict[symptom]] = 1

    # Get probability distribution across all diseases
    pred_proba  = model.predict_proba([input_vector])[0]
    pred_class  = np.argmax(pred_proba)
    disease     = le.inverse_transform([pred_class])[0]
    confidence  = round(pred_proba[pred_class] * 100, 2)

    return disease, confidence, pred_proba

Notice we use predict_proba instead of predict. This gives us a probability distribution over all 41 disease classes, not just the winning prediction. The confidence score (e.g., "87.3% confident") comes from the highest probability in that distribution.

def getSeverityDict():
    with open('MasterData/symptom_severity.csv') as csv_file:
        for row in csv.reader(csv_file):
            try:
                severityDictionary[row[0]] = int(row[1])
            except:
                pass  # silently skip malformed rows

The bare except: pass is a pragmatic choice — CSV files from the wild often have blank trailing rows or header rows that would crash on int() conversion. In production you'd want to log these silently skipped rows.

8. The Conversational Loop — Putting It All Together

The chatbot() function orchestrates the entire user experience. It follows a deliberate three-phase conversation design:

1

Context Gathering

Collects name, age, gender, symptom description, duration, severity scale (1–10), pre-existing conditions, lifestyle factors, and family history. This data isn't used by the ML model directly, but builds clinical context and empathy.

2

Initial Prediction & Guided Questions

After the first NLP extraction, a preliminary prediction is made. The chatbot then identifies the top 8 symptoms associated with that predicted disease that the user hasn't mentioned yet, and asks yes/no questions to confirm or add them.

3

Final Diagnosis Output

With the enriched symptom list, a final prediction is made and presented with: disease name, confidence percentage, descriptive paragraph, and 4 actionable precautions.

# Pull disease-specific symptoms from training data
disease_symptoms = list(
    training[training['prognosis'] == disease]
    .iloc[0][:-1].index[
        training[training['prognosis'] == disease]
        .iloc[0][:-1] == 1
    ]
)

# Ask about at most 8 symptoms not yet confirmed
asked = 0
for sym in disease_symptoms:
    if sym not in symptoms_list and asked < 8:
        ans = input(f"👉 Do you also have {sym.replace('_',' ')}? (yes/no): ")
        if ans == "yes":
            symptoms_list.append(sym)
        asked += 1

This guided Q&A approach is powerful. It simulates what a doctor does — after a preliminary hypothesis, they probe specifically for confirming symptoms. Each "yes" strengthens the model's confidence in the initial prediction, or in rare cases, could shift it entirely to a different disease.

10. Key Takeaways

This project packs a surprising number of real-world engineering lessons into a relatively compact codebase. Let's summarize what makes it instructive:

🎯

Multi-pass NLP

Combining synonyms + exact match + fuzzy matching creates a robust extractor far more resilient than any single strategy alone.

🌲

Probabilistic Output

Using predict_proba over predict unlocks confidence scores and the possibility of showing multiple candidate diseases.

🔄

Iterative Refinement

Making an initial prediction, then asking targeted follow-up questions mirrors real clinical reasoning elegantly.

📦

Separation of Concerns

Knowledge (CSVs), ML model, NLP, and UX are cleanly separated — each can be upgraded independently.

Building healthcare AI responsibly means being honest about what your model can and cannot do. A confidence score of 87% still means a 13% chance of being wrong — and in medicine, that matters enormously.
